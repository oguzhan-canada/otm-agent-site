Paper: Evaluation Methodology

Draft methodology section for the OTM-Agent technical paper. Covers experimental design, benchmark protocols, statistical methods, and reproducibility.

1. Experimental Design

1.1 Out-of-Sample Protocol

All evaluation uses a strict temporal holdout. The cutoff date (May 1, 2025) was chosen because every underlying model was trained exclusively on pre-2025 data. The 12+ month holdout ensures that no headline number can be inflated by data leakage.

Enforcement operates at three levels:

  1. Static analysis (CI-blocking) — AST parsing of all Python and SQL files catches training-table queries without date filters at build time
  2. Runtime enforcement — TrainingBQClient wrapper raises OOSLeakageError on dynamically constructed queries
  3. Causal filtering — query_ts parameter threading prevents memory retrieval from accessing future information during replay

1.2 Reproducibility

All random operations use random_state=42 across 22 model constructors. Benchmark runners accept explicit seed lists (default: [42, 43, 44, 45, 46]) for multi-seed evaluation. Bootstrap CIs use n_bootstrap=10,000 with seed 42.

Model version is tagged as v0.2.0-oos-clean — the version trained from scratch after the OOS audit.

2. Benchmark Protocols

2.1 OTM-Persona-Align (n=50 queries × 5 seeds = 250 samples)

Protocol: Each query is run twice — once with persona seeded to execution_oriented, once to analysis_oriented. For each pair, we compute:

  • Cosine distance between 64-d response embeddings (higher = more different)
  • Jaccard distance between skill sets invoked (higher = more different)

2.3 OTM-Mem-Utility (n=500 queries × 5 seeds)

Protocol: Counterfactual design. 500 queries across 5 categories (historical comparison, pattern recognition, factual domain, real-time, reasoning) run with and without memory retrieval. LLM-as-judge (blind, 0–5 scale) rates both responses.

2.4 OTM-OOS-Replay (261 trading days)

Protocol: Walk day-by-day through 2025-05-01 to 2026-05-01. For each day t:

  1. Set query_ts = t (causal filter active)
  2. Ask agent: "What is the call for tomorrow?"
  3. Record direction (long/short) and confidence
  4. Compare against actual next-day price movement

Primary metric: Directional accuracy (not Sharpe). Sharpe is secondary.

4. Honest Reporting

All results are published as they emerge, including negative findings:

  • Initial backtest Sharpe of −0.10 is reported with full root cause analysis
  • Spread forecaster and EIA surprise predictor show only marginal improvement (1–2 pp) over random baselines — acknowledged as difficult to distinguish from noise
  • OOS replay directional accuracy is modest — reported exactly as measured
  • The audit finding of 20+ unfiltered queries is published with full narrative

5. Limitations

  • Benchmark runners currently use simulated agent responses; production integration pending
  • LLM-as-judge scoring is simulated; production uses actual Sonnet evaluation
  • OOS replay uses simulated price data; production reads from BigQuery WTI prices
  • Sample sizes for some benchmarks may be insufficient for definitive conclusions
  • Persona alignment measures differentiation, not quality — different responses could both be poor
  • Memory utility is measured in controlled conditions; real-world memory is populated over time
