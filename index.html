What is OTM-Agent?

Commodity analysts integrate satellite imagery, market microstructure data, and econometric models to form views on global oil storage. This workflow demands expertise spanning remote sensing, time-series analysis, and energy trading — a combination rarely found in a single practitioner.

OTM-Agent addresses this by wrapping an operational three-stage pipeline — Sentinel-1 SAR processing, feature engineering with market data, and signal generation — with an LLM-based planner that selects which pipeline components to run and in what order, a persistent memory system that maintains context across sessions, and a persona model that adapts behavior to individual analysts.

The LLM serves as a planner, not a predictor. The statistical models in Stages 1–3 handle prediction; the LLM decides what to run, when, and how to interpret the results in context.

Key Contributions

Typed Skill Library

20 skills with declared preconditions, side effects, failure modes, and retention probes. The skill registry validates plans against a dependency DAG before execution.

OOS Holdout Enforcement via Static Analysis

All training data restricted to dates before May 1, 2025. A pytest-based CI gate uses AST parsing to scan all training queries for the holdout filter. An audit identified 20+ queries lacking the filter — all corrected before any model training.

Causal Leakage Prevention

A query_ts parameter threaded through all 8 memory retrieval paths prevents the agent from accessing future information during OOS replay. Enforced by 16 dedicated tests.

Multi-Objective Reward

R = R_directional + α·R_task + β·R_memory + γ·R_consistency + δ·R_retention − λ·C_cost. Directional alignment with price movement provides the environment-grounded signal; auxiliary components capture task quality, memory utility, and knowledge retention.
